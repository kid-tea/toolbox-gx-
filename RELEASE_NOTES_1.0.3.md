# Toolbox v1.0.3

发布日期：2026-07-03

## 主要更新

- 优化磁盘分析可视化：将原来的饼图升级为类似 WizTree 的块状矩形图，更适合查看文件夹和文件占用比例。
- 改进磁盘分析交互：点击块状图时会联动定位到对应文件夹/文件，减少在列表和图形之间来回查找的成本。
- 强化批量重命名：在前后缀基础上补充整体重命名、格式化重命名等更实用的规则能力。
- 扩展 C 盘清理能力：增加更多安全扫描项，并支持“删除前年龄过滤”，未勾选时不会按年龄条件清理。
- 优化清理页体验：减少重复选项展示，调整工具栏布局和列表列宽，界面更紧凑清晰。

## 内存释放

- 修复“清空待机列表”报 `NtSetSystemInformation 0xC0000061` 的问题，调用前会启用必要的 `SeProfileSingleProcessPrivilege`。
- 修复“清空系统缓存”和“清空待机列表”实际使用同一命令的问题。
- “清空待机列表”现在使用 `MemoryPurgeStandbyList`。
- “清空系统缓存”现在使用 `MemoryFlushModifiedList`。
- 为待机列表和系统缓存操作补充副作用说明与建议使用频率，避免用户频繁误用。

## 注册表清理

- 增强注册表备份可靠性：备份现在递归导出子键，删除整键前可恢复的信息更完整。
- 新增备份 `.sha256` 校验文件，恢复前可以发现备份是否被修改或损坏。
- 修复 `HKLM_WOW64` 启动项残留无法正确备份/修复的问题。
- 修复前会再次验证问题是否仍然存在，避免扫描后状态变化导致误删。
- 空注册表键、无效 COM/OLE 默认不勾选，降低误删风险。
- 新增“全选分类”和“安全默认”按钮：需要时可一键包含空注册表键和 COM/OLE；也可快速恢复保守扫描配置。
- 空键扫描跳过 Microsoft、Classes、Windows 等高风险区域。
- 增强命令路径解析，减少 `rundll32.exe`、`msiexec.exe`、未加引号的 Program Files 路径被误判。

## UI 与体验

- 全局启用更稳定的像素对齐和布局取整，减少 WPF 界面模糊和细微错位。
- 优化部分页面按钮、提示、状态文案，让风险操作前的信息更明确。
- 改进磁盘图形中文本拥挤问题，小文件更适合以块状方式聚合展示。

## 安全说明

- 注册表清理仍建议以保守方式使用：先扫描、查看条目、确认备份，再修复。
- 内存释放功能需要管理员权限；它不会删除文件，但可能造成短时间卡顿或缓存重新建立。
- C 盘清理默认偏安全，涉及系统缓存、驱动旧版本等项目时请根据提示确认。

## 文件

- 推荐发布文件：`publish/Toolbox.exe`
- 运行环境：Windows，.NET 8 Desktop Runtime
